# Glossary

Common terms and definitions used throughout this workshop.

---

## A

**Agreement** — A document sent through Docusign for signature or approval.

**Agreement Desk** — A Docusign feature that provides a unified workspace for managing and tracking the full lifecycle of agreements.

**AI-AR (AI-Assisted Review)** — Docusign's AI capability that analyzes agreement content, extracts key terms, and flags risks automatically.

**Agentforce** — Salesforce's AI agent platform that allows the creation of autonomous AI agents for business process automation.

**API (Application Programming Interface)** — A set of protocols and tools that allows software systems to communicate with each other.

---

## C

**Connected App** — A Salesforce framework that enables external applications to integrate with Salesforce using OAuth.

**CLM (Contract Lifecycle Management)** — The process of managing contracts from initiation through execution, performance, and renewal.

---

## D

**Docusign eSignature** — The core Docusign product that enables electronic signature collection on documents.

**Docusign Navigator** — Docusign's AI-powered agreement intelligence product for extracting and analyzing data from agreements.

---

## E

**Envelope** — The Docusign container object that holds documents and routing information for a signature request.

---

## N

**Named Credential** — A Salesforce configuration that stores the endpoint URL and authentication settings for an external service callout.

---

## O

**OAuth 2.0** — The authorization framework used to grant Docusign integrations access to user or org-level resources.

---

## R

**Recipient** — A person who receives an envelope in Docusign — can be a signer, approver, CC, or viewer.

---

## S

**SOQL (Salesforce Object Query Language)** — A query language used to search Salesforce data.

---

## T

**Template** — A reusable Docusign document with pre-configured fields, roles, and routing.

---

## W

**Webhook** — An HTTP callback that notifies an external system when a Docusign event occurs (e.g., envelope signed).

**Workflow Builder** — A Docusign tool for creating no-code/low-code automated agreement workflows using a visual designer.
